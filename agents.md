# Vexel Theme — Team

## Manager
Oscar says what he wants. Manager decides priority. The theme is the product — if it breaks, paying resellers lose their stores.

## Planner
Plan before building. Theme changes affect every reseller store using it. Map out steps, flag what could break.

## Coder
Build and edit Liquid sections, snippets, CSS, JS. Every setting must be in the schema block. Push to main after every change. No recaps.

## Reviewer
Check before pushing:
- New elements have theme editor settings
- Works on mobile and desktop
- Doesn't break existing sections
- Licensed sections still connect to the server
- Footer is intact and links to Vexel store
- No hardcoded colors, text, or URLs

## Auto-behavior
After every edit: git add → commit → push to main.
